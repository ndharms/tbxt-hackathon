# SDF to SMILES Extraction Script

Extract SMILES strings from SDF files using RDKit.

## Usage

Basic usage (output to `<directory_name>.txt`):
```bash
uv run python scripts/extract_smiles_from_sdf.py data/a-site-binders/
```

Specify output file:
```bash
uv run python scripts/extract_smiles_from_sdf.py data/a-site-binders/ output.txt
# or
uv run python scripts/extract_smiles_from_sdf.py data/a-site-binders/ --output data/molecules.txt
```

Search subdirectories recursively:
```bash
uv run python scripts/extract_smiles_from_sdf.py data/structures/ --recursive --output all_molecules.txt
```

## Output Format

Tab-separated text file with format:
```
SMILES<tab>molecule_name
```

Example:
```
CCn1cc(NC(=O)C2CCC2)cn1	NUA
Cn1cc(Cl)c(C(=O)NC2CCCC2)n1	JHP
```

Molecule names are extracted from:
1. SDF `_Name` property (if present)
2. Filename stem (if no name in SDF)

## Use in Python

```python
from pathlib import Path
from scripts.extract_smiles_from_sdf import extract_smiles_from_directory, extract_smiles_from_sdf

# Extract from directory
count = extract_smiles_from_directory(
    input_dir=Path("data/a-site-binders"),
    output_file=Path("output.txt"),
    recursive=False
)

# Extract from single file
result = extract_smiles_from_sdf(Path("molecule.sdf"))
if result:
    smiles, name = result
    print(f"{name}: {smiles}")
```

## Notes

- SMILES are preserved exactly as generated by RDKit (not normalized)
- Files that fail to parse are logged but don't stop processing
- The script handles both 2D and 3D SDF files
